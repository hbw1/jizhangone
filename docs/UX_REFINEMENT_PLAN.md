# 鸿运账本 UX 改造方案

## 背景

当前版本已经具备基础可用性：

- 本地记账 CRUD
- 一句话记账
- 统计报表
- 备份导入导出
- 云端登录与同步 Beta

但页面层级和信息密度还不够克制，主要问题是：

- 首页内容过多，不够像“打开就记账”的主界面
- 账本操作、最近记录、次要管理项占据了首页主视野
- 报表页首屏重点不够明确
- 设置页承担了太多直接操作，像“工具仓库”，不够清晰
- 图标风格不够简约，也不够有辨识度

本轮改造目标不是新增功能，而是把现有功能重新组织成更清晰的体验。

## 设计原则

### 1. 首页只做三件事

用户打开首页，第一眼只需要看到：

- 当前账本
- 本月结余、收入、支出
- 记账入口

首页不再承担：

- 最近记录展示
- 大量账本管理操作
- 复杂说明文案

### 2. 高频操作前置，低频操作下沉

高频：

- 记一笔
- 一句话记账
- 看本月收支

低频：

- 新建账本
- 账本改名
- 账本切换
- 成员管理
- 账户管理
- 分类管理

低频操作统一放到二级入口里，不占据首屏主区域。

### 3. 报表页首屏必须先回答“这个周期花了多少”

用户进报表页，先看清：

- 当前周期
- 总收入
- 总支出
- 结余/净值

时间筛选必须明显，但不能喧宾夺主。

### 4. 设置页只做导航，不直接堆太多操作

设置页应该像“管理中心”：

- 云端同步
- 成员管理
- 账户管理
- 分类管理
- 本地备份

每个模块进入二级页面后再进行新增、改名、停用等具体动作。

### 5. 视觉方向统一为“简约喜庆”

风格方向：

- 红色为主基调
- 文案减少
- 信息块减少
- 一屏一个重点
- 图标简约，不做复杂拟物

## 页面改造方案

## 首页

### 保留

- 当前账本名称
- 本月结余
- 本月收入
- 本月支出
- `记一笔`
- `一句话记账`

### 移除

- 最近记录
- 账本改名按钮
- 新建账本按钮
- 其他次要说明信息

### 新结构

- 顶部：左上角菜单按钮 + 当前账本名
- 中部：结余主数字
- 下部：收入/支出双卡
- 底部主动作：`记一笔`、`一句话记账`

### 二级菜单内容

左上角三横线菜单内放：

- 切换账本
- 新建账本
- 账本改名
- 进入设置

## 记账页

### 目标

继续保持简约，但入口更明确。

### 结构

- 顶部标题：记账
- 双入口切换：
  - 一句话记账
  - 直接记账

### 要求

- 保持少文字
- 不堆解释
- 用户一眼知道该怎么开始输入

## 明细页

### 调整

把首页上的“最近记录”完整收回明细页。

### 明细页承担

- 全部流水展示
- 筛选
- 编辑
- 删除

### 方向

首页不再展示流水，明细页成为唯一流水主入口。

## 报表页

### 首屏重点

用户一进来必须先看到：

- 当前周期名称
- 收入
- 支出
- 结余

### 时间筛选要求

筛选保留：

- 本月
- 上月
- 近 30 天
- 本年
- 自定义

但放在更紧凑、更直观的位置，不抢主数据风头。

### 报表展示顺序

1. 周期总览
2. 收入/支出概览
3. 分类占比
4. 成员统计
5. 账户统计

## 设置页

### 现状问题

当前设置页信息过多，操作和展示混在一起，不够清晰。

### 新结构

设置页只保留这些一级入口：

- 云端同步
- 成员管理
- 账户管理
- 分类管理
- 本地备份
- 关于应用

### 二级页面拆分

- 云端同步页
- 成员管理页
- 账户管理页
- 分类管理页
- 备份页

这样一级页面更清楚，也更适合继续扩展。

## 图标改造方向

当前图标需要更简约、更有识别度。

推荐方向：

- 一个极简折线向上图形
- 类似“上涨”“进账”“趋势走高”的视觉隐喻
- 几何形、少细节、扁平化
- 保留红金体系，不要复杂阴影

不建议：

- 过于写实的钱包、硬币、账本图形
- 图形元素太多
- 颜色过杂

## 分批实施方案

## Phase 1：信息架构重排

目标：

- 首页变简约
- 最近记录移出首页
- 账本管理下沉到二级菜单
- 设置页一级入口化

具体工作：

- 重做首页结构
- 增加首页左上角菜单
- 精简首页文案
- 从首页删除最近记录
- 设置页改为列表入口页

验收标准：

- 首页首屏只剩账本、收支结余、两个记账入口
- 用户能在 3 秒内找到记账入口
- 用户能在菜单中找到账本管理

## Phase 2：记账主流程优化

目标：

- 记账页双入口更清晰
- 直接记账和一句话记账切换更明确

具体工作：

- 重排记账页顶部结构
- 减少说明文案
- 优化表单主次关系

验收标准：

- 用户进入记账页后不需要解释也知道怎么记
- 记一笔和一句话记账都能一眼区分

## Phase 3：报表首屏重构

目标：

- 报表页第一屏更像“收支看板”

具体工作：

- 重排报表首屏数据层级
- 把周期筛选做得更紧凑
- 统一下面各报表模块的视觉样式

验收标准：

- 用户一进来能立刻说出“这个周期花了多少”
- 筛选入口明显，但不遮挡主数据

## Phase 4：设置页二级管理

目标：

- 设置页更清晰、更轻

具体工作：

- 拆分成员管理页
- 拆分账户管理页
- 拆分分类管理页
- 拆分备份页

验收标准：

- 设置首屏不再出现过长的操作列表
- 一级页和二级页职责清晰

## Phase 5：图标与视觉收口

目标：

- 图标更高级
- 整体视觉更统一

具体工作：

- 重绘 launcher 图标
- 统一首页、报表、设置页面的组件语言
- 控制页面文字量和色块密度

验收标准：

- 图标在桌面上有辨识度
- 页面更简洁，不显乱

## 建议实施顺序

优先级建议：

1. 首页
2. 设置页结构
3. 记账页
4. 报表页
5. 图标收口

原因：

- 首页和设置页最影响日常感受
- 这两块先改完，整体产品气质会先稳定下来
- 之后再处理报表和图标，收益更集中

## 是否涉及服务端

本轮 UX 改造默认只涉及 Android App 端。

不涉及：

- 后端接口
- 数据库结构
- 同步协议
- 服务器部署

因此默认不需要 OpenClaw 介入。
